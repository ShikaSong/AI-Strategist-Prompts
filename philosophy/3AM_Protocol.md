# The 3 AM Evolution Protocol (3AM 进化协议)

## 什么是 3 AM 协议？
在生物学中，人类的海马体（Hippocampus）会在深度睡眠期间将短期记忆回放并传输给大脑皮层（Neocortex），从而形成长期记忆。

普罗米修斯架构模仿了这一过程。我们不进行实时的全量计算（昂贵且低效），而是选择在用户当地时间的 **03:00 AM** 触发一次深度的系统级复盘。

## 为什么是异步的？
1.  **降噪 (De-noising)**: 白天的数据（L1 感官流）是嘈杂的。只有在一天结束后，AI 才能以上帝视角审视全天，判断哪些是噪音，哪些是信号。
2.  **成本控制**: 深度推理（Reasoning）非常消耗 Token。通过每日一次的批处理，我们能用最强的模型（如 GPT-4o 或 Gemini Pro）做最深度的思考，而日常交互使用快速模型。

## 协议执行流程 (The Pipeline)
1.  **提取 (Extraction)**: 扫描昨日所有的 Logs, Tasks, Transactions。
2.  **推演 (Foresight)**: 寻找因果关系。例如：“用户昨天下午喝了咖啡，导致昨晚睡眠分数下降” -> 生成新知识 `[Fact: Caffeine Sensitive]`.
3.  **仲裁 (Arbitration)**: 如果新知识与旧规则冲突（例如旧规则说用户喜欢咖啡），根据时间戳权重，更新 L4 核心规则库。
4.  **指令 (Directives)**: 生成次日的 `Planning Directives`（例如：“今日禁止安排 14:00 后的咖啡会议”）。

这就是普罗米修斯“越用越懂你”的底层机制。
